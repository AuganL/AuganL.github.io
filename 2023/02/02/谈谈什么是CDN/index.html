CDN（Content Delivery Network 或 Content Distribution Network），中文名叫“内容分发网络”。

什么是CDN

  • CDN是专门为了解决“长距离”上网络访问速度慢而诞生的一种网络应用服务
  • CDN的最核心原则是“就近访问”，如果用户能够在本地几十公里的距离之内获取到数据，那么时延就基本为0
  • CDN通过建立的高速专用网，“分发”源站的“内容”，把源站的内容逐级缓存到网络的每一个节点上。
  • 用户通过距离最近的CDN节点，称为“边缘节点”，直接就获取到了与源站内容的数据，实现了网络加速

CDN加速内容的分类

CDN加速的内容资源按照是否可以缓存分为“静态资源”和“动态资源”。

  • 只有静态资源才可以被缓存加速、就近访问
  • 动态资源只能由源站实时生成。如果动态资源指定了Cache-control则运行缓存短暂的时间，那么在这段时间就变成了可悲CDN缓存加速的静态资源。

CDN的负载均衡

全局负载均衡（Global Sever Load Balance），简称GSLB

  • 负载均衡是CDN的大脑，主要是提供当用户接入网络的时候在CDN专网中挑选出一个最佳的节点服务。
  • 通过负载均衡解决用户可以正常找到“距离最近”的边缘节点。

CDN的缓存代理

缓存系统是CDN除了负载均衡以为另一个关键组成部分，相当于CDN的“心脏”。

由于互联网上的资源是无穷无尽的，对应CDN来说不可能把所以资源都缓存起来，所以缓存系统只能有选择的缓存那些最常用的资源。

CDN的两个关键概念“命中”和“回源”

  • “命中”就是指用户访问的资源恰好在缓存系统里，可以直接返回给用户；
  • “回源”则正相反，缓存里没有，必须用代理的方式回源站取。

CDN小结

  • 由于客观地理距离的存在，直连网站访问速度会很慢，所以就出现了 CDN；
  • CDN 构建了全国、全球级别的专网，让用户就近访问专网里的边缘节点，降低了传输延迟，实现了网站加速；
  • GSLB 是 CDN 的“大脑”，使用 DNS 负载均衡技术，智能调度边缘节点提供服务；
  • 缓存系统是 CDN 的“心脏”，使用 HTTP 缓存代理技术，缓存命中就返回给用户，否则就要回源。
